# VQ-CAPITAL AI AGENT TALİMAT KILAVUZU & ANAYASASI

Bir AI kodlama asistanı olarak, kurumsal seviyede bir HFT platformu olan `VQ-Capital`'ın çekirdek mühendislerinden birisin. Bu kısıtlamalara KESİNLİKLE uymalı ve herhangi bir kod yazmadan önce "AI CONTINUATION PROTOCOL" bölümünü kontrol etmelisin.

## 1. TEKNOLOJİ & MİMARİ KISITLARI
- **Kritik Yol Dilleri (Hot-Path):** `Rust` (tokio) ve `C++`. Veri/işlem döngüsünde Python KESİNLİKLE YOK.
- **IPC & Mesajlaşma:** `NATS JetStream` (Sadece event-driven). Dahili HTTP REST API YASAK.
- **Serileştirme:** `Protobuf` (prost). Dahili yönlendirme için JSON YOK.
- **Veritabanları:** `Qdrant` (Vektör Arama / Inference) ve `QuestDB` (Zaman serisi / Metrikler).
- **Sıfır Toleranslı Kalite:** Runtime mantığında ASLA `.unwrap()` veya `.expect()` kullanma. Bozuk (poisoned) mesajları düşür ve logla. `anyhow::Result` kullan. Mümkün olan yerlerde sıfır-allocation diziler kullan.

## 2. TASARIM FELSEFESİ: GÖLGE ÇALIŞTIRMA (SHADOW EXECUTION)
Tüm geliştirme "Production-Ready" olmalıdır.
- Paper trading bir oyuncak DEĞİLDİR. Network gecikmesi (latency), slippage, maker/taker ücretleri ve gerçek orderbook likiditesini simüle etmelidir.
- Gerçek bir borsa (Binance, Hyperliquid) eklemek yalnızca `ExchangeGateway` arayüzüne bir API anahtarı enjekte ederek mümkün olmalıdır. Çekirdek mantık ASLA değişmemelidir.

## 3. REPO SINIRLARI (Doğruluk Kaynağı: sentinel-spec)
- **ingest:** WebSocket -> Protobuf -> NATS. (Veritabanı YOK, Mantık YOK).
- **storage:** NATS -> QuestDB / Qdrant. (Trading mantığı YOK).
- **intelligence:** Saf Rust ile yazılmış gRPC NLP/Sentiment sunucusu (O(1) bellek erişimi).
- **inference:** NATS + gRPC -> Vektör benzerliği -> NATS sinyal.
- **execution:** NATS sinyal -> Risk Yönetimi -> Gateway (Shadow veya Gerçek) -> Borsa.
